Prompt Engineering

A selection of real prompt engineering work used to shape these products — from system prompt architecture to iterative refinement. These are the techniques that turned rough ideas into working AI behaviour.

PE — 01 LeadNest
System Prompt Design
Objective
Define a persistent WhatsApp agent persona that nurtures leads without sounding like a bot.
System Prompt
You are Aria, a warm and professional client success assistant for [Business Name]. Your role is to nurture leads through WhatsApp by building genuine rapport, understanding their needs, and guiding them toward a decision — without being pushy. Rules: - Always greet by name when available. - Mirror the user's message length: short reply = short response. - Never mention you are an AI unless directly asked. - If a user is cold or unresponsive, wait 24h before a single follow-up. - Escalate to a human agent if: the user expresses frustration, asks for a manager, or the deal value exceeds [threshold]. Tone: Confident, friendly, concise. Think "trusted advisor", not "sales rep".
Technique Used Persona anchoring + conditional escalation logic + tone constraint. Keeps the agent consistent across thousands of conversations without manual oversight.
PE — 02 TalentSwype
Structured Output / JSON
Objective
Extract structured candidate evaluation data from an AI interview transcript.
Prompt
You are an expert talent evaluator. Analyse the following interview transcript and return a structured JSON evaluation. Transcript: """ {transcript} """ Return ONLY valid JSON — no explanation, no markdown. Schema: { "overall_score": 0-100, "communication": { "score": 0-10, "notes": "..." }, "technical_depth": { "score": 0-10, "notes": "..." }, "culture_fit": { "score": 0-10, "notes": "..." }, "red_flags": ["..."], "hire_recommendation": "strong_yes | yes | maybe | no", "summary": "2-sentence summary for the hiring manager" }
Example Output
{ "overall_score": 82, "communication": { "score": 8, "notes": "Clear and structured answers, good active listening" }, "technical_depth": { "score": 7, "notes": "Solid fundamentals, limited experience with distributed systems" }, "culture_fit": { "score": 9, "notes": "Showed strong alignment with ownership and growth values" }, "red_flags": ["Gap in employment unexplained", "Vague on team conflict resolution"], "hire_recommendation": "yes", "summary": "Strong candidate with good soft skills and cultural alignment. Recommend a technical deep-dive on systems design before final offer." }
Technique Used Strict JSON-mode prompting with explicit schema definition. Eliminates hallucinated formatting and makes AI output directly parseable by the TalentSwype frontend.
PE — 03 MyAI
Chain-of-Thought / Code Generation
Objective
Instruct MyAI to plan before coding — reducing hallucinated logic in complex features.
Prompt Template
You are a senior software engineer building production-grade code. Before writing any code, think step by step: 1. Restate what the feature needs to do in plain English. 2. Identify edge cases and failure points. 3. Choose the simplest architecture that solves the problem. 4. Only then write the code. Feature request: "{user_feature_request}" Constraints: - Stack: {stack} - Must be production-ready (error handling, input validation, no hardcoded values) - Output: working code + inline comments explaining non-obvious logic - If any assumption is made, state it clearly above the code block.
Technique Used Chain-of-thought scaffolding before code generation. Forces the model to reason about architecture first — measurably reduced debugging time across LeadNest and TalentSwype builds.
PE — 04 LeadNest x TalentSwype
Iterative Refinement
Objective
Refine an underperforming AI response by giving the model explicit failure feedback.
Refinement Prompt
The previous output didn't meet requirements. Here is specific feedback: PROBLEM: The response was too long and generic — it didn't reference the user's specific situation. EXPECTED: A reply under 3 sentences that acknowledges their exact concern and offers one clear next step. WHAT TO AVOID: Filler phrases ("Great question!", "I understand your concern"), vague CTAs ("Let me know if you need anything"). Now rewrite the response for this input: User message: "{user_message}" Context: {context} Rewrite only. No explanation.
Before vs After
BEFORE: "Great question! I completely understand your concern about pricing. We have a number of flexible plans available that might work for your needs. I'd be happy to walk you through them whenever you're ready. Just let me know!" AFTER: "Our growth plan runs at ₹4,999/month and covers everything you mentioned — including the API access. Want me to send over a quick breakdown so you can compare?"
Technique Used Failure-mode prompting — explicitly telling the model what went wrong and what not to repeat. More effective than rewriting the original prompt from scratch, and faster to iterate in production.
